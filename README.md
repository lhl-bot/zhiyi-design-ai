# AI赋能服装定向设计 V2 工作台

这是根据已审批的《AI赋能服装定向设计》启动方案落地的内部 Web 工作台，覆盖首期 9 家核心客户风格档案、数据沉淀、AI 定向出款、参考图融合、结果筛选标注和导出。

## 本地启动

> **环境要求：Node.js ≥ 20.6**（`npm run dev:api` 使用了 `node --env-file`，低版本会报错）。

需要同时起两个进程：前端工作台 + ERP 只读后端。

```bash
npm install

# 1) ERP 只读后端（连接信息通过环境变量传入，绝不写进代码）
ERP_DB_HOST=123.57.4.44 ERP_DB_PORT=2531 \
ERP_DB_DATABASE=RichMaxGarmentBF_ssmj \
ERP_DB_USER=只读账号 ERP_DB_PASSWORD=只读密码 \
npm run dev:api

# 2) 前端（另开一个终端）
npm run dev
```

打开终端给出的本地地址，默认是 `http://localhost:5173/`。前端的 `/api/erp/*` 会被 Vite 代理到 `:8787` 的只读后端。

> 安全提醒：示例里用的是 `sa` 账号，正式使用请改成**只读专用账号**，并把密码放在环境变量或 `.env`（已在 `.gitignore` 思路内，勿提交）。

## 当前实现范围

界面是「一屏极简流程」：从上到下三步走完，不需要切换页面。

1. **选客户**（右上角下拉）→ 自动读取该客户 ERP，命中真实订单时**自动**生成并显示真实画像（只读，不需要手动编辑）：
   - 已命中真实订单数据：JRC、TJ、HUP、NEX、PWT、VG
   - 自动生成：客户全称/等级/年采购、风格标签（产品品类+主力面料）、面料偏好、去重后的色彩偏好、单件价格带（合同总额÷报价数量）、订单趋势、大货/样品返单占比
   - 4F：ERP 有主数据但无成衣订单，卡片提示需业务确认下单主体，用示例画像兜底
   - SEWD / DVH：当前简称未命中 ERP，提示需业务/设计部提供「简称→ERP 全称」映射
   - **品牌情报（公开信息）**：内置 4F/TJ/PWT/NEX/HUP/VG 等真实品牌的公开定位、风格、品类、趋势，免 Key 即时显示
   - **AI 联网智能分析**（可选）：点按钮可用 **Agent-Reach / Tavily / 混合情报**采集品牌官网与公开网页，再用 **DeepSeek** 结合 ERP 数据生成品牌定位、（带来源的）当季趋势判断和 3-5 条具体出款方向；需在「高级」配置 `DEEPSEEK_API_KEY`（或用环境变量启动后端），Tavily 模式另需 `TAVILY_API_KEY`。默认 `deepseek-v4-flash`（约数秒），可用 `DEEPSEEK_MODEL=deepseek-v4-pro` 换更深度分析。Agent-Reach 命令未安装时，会先退化到零配置网页读取。
2. **生成定向款**：设数量+季节，点一个大按钮出图；也可上传参考图生成融合款。
   - **真实出图**：在「高级」选「火山方舟 Doubao 真实出图」并配置 `ARK_API_KEY`，用 Doubao Seedream **逐款单独**生成真实款式图（每款一张、提示词聚焦，画质与区分度更高），陆续返回；参考图融合走图生图。需先在火山方舟控制台**开通对应 Seedream 模型**（默认 `doubao-seedream-5-0-260128`），`ARK_MODEL` 与开通的 Model ID 一致。
   - **图片会永久保存**：火山方舟原始图片 URL 仅 24 小时有效，后端会自动把每张图下载落地到 `server/generated/`，并通过 `/api/img/...` 提供永久链接，刷新或隔天都不会丢。
   - 每张图卡右上角有**下载**和**删除**按钮，大图弹窗里也能下载；删除会同时清理服务器上的图片文件。
3. **挑款 / 导出**：每款带综合分/趋势分/商业分/成本/关键细节/设计师备注，点「选入精修池」挑款，导出 Markdown 推款包或 JSON。

- 右上角「高级」里收纳次要设置：生成接口（本地模拟 / Sendance）、创意强度、必须保留/避免项、手动重新读取 ERP。
- 本地持久化：画像、生成记录、接口配置自动保存在浏览器 localStorage。

## 后续真实接口替换点

- `src/services/generation.ts`：当前为本地模拟生成，可替换为 Sendance 2.0 API 或内部图像生成接口
- `src/data/customers.ts`：替换为 ERP、历史款式图、客户官网和电商/社媒采集结果
- `src/utils/export.ts`：可改为后端生成 Excel、PDF 或推款包

## 当前限制

当前版本不包含真实 Sendance 2.0 API 调用，因为还没有接口地址、鉴权方式和 API Key。界面中已预留 Sendance 配置入口，拿到接口文档后优先替换 `generateLooks` 的本地模拟逻辑。

## 数据安全默认口径

当前原型只使用本地样例数据。上线时建议客户款式图、订单数据和生成记录存储在公司内网；如需云端推理，只上传最小必要信息或匿名化特征。
